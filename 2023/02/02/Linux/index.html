第四章 基础篇-目录结构

目录结构介绍

基本介绍：

  1. 树状层级结构，最上层是根目录 /
  2. 实际生产是没有界面的，要记住目录结构
  3. 在Linux一切皆文件。硬件也是映射成文件管理

目录结构详解

/bin 常用 （/usr/bin、/usr/local/bin）

是Binary的缩写，存放着最常用的指令

/sbin （/usr/sbin、/usr/local/sbin）

s就是superuser，存放系统管理员使用的系统管理程序

/home 常用

存放普通用户的主目录，每个用户有一个自己的目录，目录名用账号命名

/root常用

系统管理员，即超级权限者的用户主目录

/lib

系统开机所需的最基本的动态链接共享库，作用类似windows里的DLL文件。几乎所有 应用程序都需要这些共享库

/lost+found一般是空的，但是是隐藏的

/etc常用

所有的系统管理所需的配置文件和子目录，如安装mysql数据库 my.conf

/usr 常用

用户的很多应用程序和文件放在这，类似windows下的program files

/boot 常用

存放启动时使用的核心文件，包括链接问价和镜像文件

/proc 【不能动】虚拟的，系统内存的映射，访问其来获取系统信息

srv 【不能动】service的缩写，一些服务启动之后要提取的数据

/sys 【不能动】linux 2.6 的变化，其下安装了一个新出现的文件系统sysfs

/tmp存放临时文件

/dev 类似于windows的设备管理器，把所有的硬件用文件的形式存储

/media 常用 linux系统会自动识别一些设备,例如U盘、光驱等等,当识别后，linux会把识别的设备挂载到这个目录下

/mnt 常用

系统提供该目录是为了让用户临时挂载别的文件系统的,我们可以将外部的存储挂载在/mnt/…..上，然后进入该目录就可以查看里的内容了。

/opt这是给主机额外安装软件所摆放的目录。约定俗成。如安装ORACLE数据库就可放到该目录下。默认为空

/usr/local常用

这是另一个给主机额外安装软件所安装的目录。般是通过编译源码方式安装的程序

/var常用

这个目录中存放着在不断扩充着的东西，习惯将经常被修改的目录放在这个目录下。包括各种日志文件

/selinux [security-enhanced linux]

SELinux是一种安全子系统，它能控制程序只能访问特定文件，有三种工作模式,可以自行设置

第六章 实操篇-vi和vim编辑器

基本介绍

vi 即文本编辑器

Vim 具有程序编辑能力，可看作是Vi增强版，可以以字体颜色判断语法正确性。

常用三种模式

一般模式：默认的。可以使用【上下左右】移动光标，【删除字符】【删除整行】处理文档内容，【复制、粘贴】处理文件数据。

插入模式：编辑模式，输入字符。

命令行模式:提供相关指令，完成读取，存盘，替换，离开vim等操作。

基本使用

vim hello.java:开发一个Java文件

i:插入模式

按esc 输入:wq 回车。 write quit 保存退出

模式相互切换

i：按下 i、I、o、O、a、A、r、任何一个字母之后进入编辑模式，一般用 i

/：一般模式进入命令行模式

set nu：设置行号显示。set nonu：不显示行号

快捷键

yy：复制当前行。5yy：复制当前开始行的5行

p：黏贴

dd：删除当前行。5yy：删除当前开始行的5行

/hello：进入命令行，查找“hello”，n：查看下一个“hello”

G：定位到末尾一行。

gg：定位到第一行。

u：（esc后）编辑模式下的操作，执行撤回操作

输入20 按shift+g：定位到第20行。输入的行号是看不见的。

第七章 实操篇-开机重启和用户登录注销

关机与重启

shutdown -h now：立刻关机

shutdown -h 1：1分钟后关机

shutdown -r now：立刻重启

halt：立刻关机

reboot：立刻重启

sync：把内存的数据同步到磁盘

注意：

  1. 重启或者关机，首先要运行sync命令。防止比如正在编辑的文件没保存
  2. 目前shutdown、reboot、halt均已经内置了sync操作。

用户登录与注销

基本介绍：

  1. 登陆是尽量少用root登陆，避免权限大的操作失误。可以普通登陆后用su - root切换管理员身份。
  2. 在提示符下输入logout注销用户。

注意：

  1. logout图形界面级别下无法使用，需要在级别 3 使用
  2. 从普通用户到root，可以用logout退回 之前的用户登陆状态

第八章 实操篇-用户管理

基本介绍

Linux是一个多用户多任务的系统。任何用户使用系统资源都需要注册。

添加用户

语法：useradd 用户名

注意：

  1. 创建用户成功后，会自动创建和用户同名的home目录，即/home/milan。
  2. 可以通过useradd -d 指定目录 新用户名指定home目录。

指定/修改密码

语法：passwd 用户名后回车

举例：passwd milan后回车

pwd：显示当前在哪个路径下

注意：

从权限高的用户到权限低的用户不需要密码。反之需要。

删除用户

语法：

userdel 用户名 保留home目录

userdel -r 用户名 同时删除home目录。注意：实际应用时考虑员工离职后保留他的工作资料。

userdel在sbin也就是superbin下。

查询用户信息

id 用户名 ：查询信息

切换用户

在操作Linux中，如果当前用户权限不够，可以通过su -指令，切换到高权限用户，比如root

su - root

查看当前用户

who am i 或者whoami显示第一次登陆到系统的信息

用户组

类似于角色。系统可以对有共性/权限的用户方便统一管理。

groupadd 组名：增加组。没有提示信息一般就成功了。Linux没有消息就是好消息

groupdel：删除组

useradd -g 用户组 用户名：新增用户，并添加到指定用户组。不指定的话默认开一个同名的组给新用户。

usermod -g 用户组 用户名：修改用户所在的组

用户和组相关文件

/etc/passwd：用户的配置文件，记录用户的各种信息。

每行的含义：用户名：口令：用户标识号：组标识号：注释性描述：主目录：登陆Shell

shell理解为将cd之类的命令解释给内核执行的一种“解释器”。如bash

/etc/shadow：口令配置文件。

每行的含义：登录名：加密口令：最后一次修改时间：最小时间间隔：最大时间间隔：警告时间：不活动时间：失效时间：标志

/etc/group：组的配置文件，记录组的信息

每行的含义：组名：口令：组标识号：组内用户列表

第九章 实操篇-实用指令

指定运行级别

说明：

0：关机

1：单用户【找回丢失密码】

2：多用户状态 没有网络服务 基本不用

3：多用户状态 有网络服务 没有图形界面用的最多

4：系统未使用保留给用户

5：图形界面

6：系统重启

常用的是3和5，也可以指定默认运行级别。

init [0123456] 切换不同的运行级别。

CentOS7以后对级别进行了简化，主要用以下两个【其他也保留了】。在文件etc/inittab中

multi-user.targe : analogous to runlevel 3

graphical.targe : analogous to runlevel 5

systemctl get-default：查看现在的默认target

systemctl set-default TARGET.target：修改设置默认target

如何找回root密码？

不同版本不尽相同。CentOS 7.6 为例。

  1. 启动系统。在进入之前动上下键使之停留在如下界面。并在第一个按e进入编辑界面。

  1. 光标移动到“Linux16”开头的那一行，挪到最后。

  1. 输入init=/bin/sh 然后Ctrl + X。进入单用户模式
  2. 在光标闪烁位置输入 mount -o remount,rw/ 回车
  3. 新的一行，输入passwd回车，输入密码。回车，再次输入。回车。密码长度最好8位以上，但不必须。
  4. 输入touch /.autorelabel 回车
  5. 输入exec /sbin/init回车。执行时间有点长。完成后，系统自动重启，新的密码生效。

帮助指令

  1. man [命令或配置文件]：查看帮助信息。举例：man ls：查看ls命令的帮助信息。空格键“翻页”。

    从man显示信息页面退出，按q

    ls命令的选项：

    -a 列出所有文件。在Linux下，隐含文件是以 点 开头的。

    -l 单列输出。文件信息更加全面。

    -h 也就是human,把文件大小原先默认的字节单位改成人看的单位，M

    -la 选项可以组合使用使用，且没有顺序。

    可以指定目录，如 ls -al /root

  2. help ：获得shell内置命令的信息

文件目录类

  1. pwd：显示当前工作目录的绝对路径。

  2. ls [选项] [目录或者文件]：上文已经提到。

  3. cd [参数]：切换到指定目录。

  4. cd ~或者cd：回到自己的家目录。

  5. cd ..：回到当前目录的上一级目录

  6. mkdir [选项] [目录]：创建目录。-p：创建多级目录。

    举例：mkdir -p /home/animal/tiger：在home下面创建animal，其下创建tiger。

  7. rmdir [选项] [目录]：删除空目录。有内容的话删不掉。如果删除有内容的：rm -rf。recursive force

  8. touch 文件名称：创建一个空文件。

  9. cp [选项] [原文件地址] [目的地]：拷贝文件到指定目录。-r 递归复制整个文件夹。\cp：强制覆盖，不提示目的地的重名文件是否覆盖提示。

  10. rm [选项] 文件或目录：移除文件或目录。-r：递归删除整个文件夹。-f：强制删除不提示。

  11. mv：移动（剪切、粘贴）文件与目录或移动并重命名或重命名（同级目录处）。

  12. cat [选项] 要查看的文件：查看文件内容，只能查看，比vim安全。-n：显示行号。

    为了浏览方便，加上管道命令(把前面命令的结果给下一个命令)理解为组合的命令，后接| more，回车浏览更多一行。

  13. more 要查看的文件 ：是一个基于VI编辑器的文本过滤器，以全屏幕的方式按页显示文本文件内容。内置了若干快捷键：

    q：离开more，不再显示文件内容

    =：输出当前行的行号

    :f：输出文件名和当前行的行号

    空格键：向下翻一页

    回车：向下翻一行

    Ctrl + F：向下滚动一屏

    Ctrl + B：返回上一屏

  14. less 要查看的文件 ：用来分屏查看文件内容。与more类似，但是比之强大，支持各种显示终端。在显示内容时，不是一次将整个文件加载显示，而是根据需要加载，适合显示大型文件。

    空格键：向下翻一页

    pagedown：向下翻一页

    pageup：向上翻一页

    /字串：向下搜寻。n：向下查找。N：向上查找

    ？字串：向上搜寻。n：向上查找。N：向下查找

    q：离开less，不再显示文件内容

  15. echo：输出内容到控制台。

    echo $PATH：输出环境变量。或者echo $HOSTNAME：输出当前主机名。

  16. head：显示文件的开头部分内容，默认情况下显示前10行

    head -n 5 /etc/profile：查看profile前5行

  17. tail：输出文件中尾部的内容，默认情况下显示前10行

    tail -n 5 /etc/profile ：查看profile尾5行

    tail -f 文件：实时追踪该文档的所有更新。一个终端对文件监控，另一个终端对其的修改可以被捕捉到。Ctrl + C 退出

  18. >与>>：输出重定向。追加。

    ls -l [/home] >a.txt：列表的内容写入文件a.txt中（覆盖写，全清空再写入）如果没有这个文件，会创建

    ls -al >>aa.txt：列表的内容（包括隐藏的文件）追加到文件aa.txt的末尾

    cat 文件1 > 文件2：文件1的内容覆盖到文件2

    echo "内容">>文件：将’’内容’’追加到文件末尾

    cal：打印显示本月日历信息。

    cal 2020：打印显示2020的所有日历信息

  19. ln：软连接也称符号链接，类似于windows的快捷方式，主要存放链接其他文件的路径。

    ln -s /root /home/myroot：再home目录下创建一个软连接叫myroot，连接到/root目录。

    cd myroot：相当于到root目录

    rm /home/myroot：删除软连接，跟删除文件一个道理。

  20. history：查看已经执行过的历史指令，也可以执行历史指令

    history ：显示所有历史指令

    history 10：显示最近10条

    !5：执行历史编号为5的指令

时间日期类

  1. date：显示当前日期

    date "+%Y-%m-%d"：注意字母大小写。显示当前年月日。形如2022-10-27。命令重点的减号是自定义的分隔符，用空格也可以。

    date +%Y：显示当前年份

    date +%m：显示当前月份

    date +%d：显示当前日期

    date "+%Y-%m-%d %H:%M:%S"：显示年月日时分秒。形如2022-10-27 20:41:40

    date -s 字符串时间：设置系统当前时间

查找搜索类

  1. find [搜索范围] [选项]：从指定目录向下递归地遍历各个子目录，将满足条件的文件或者目录显示在终端

    find /home -name hello.txt或find /home -name *.txt：按照指定的文件名查找模式查找文件

    find /opt -user tom | more：查找属于指定用户名的所有文件。组合使用了管道

    find / -size +200M：按照指定的文件大小查找文件。查找大于200M的，小于用 减号。等于不附加符号。单位有k,M,G

  2. locate：快速定位文件路径。利用事先建立的系统中所有文件名称及路径的locate数据库实现快速定位。无需遍历整个文件系统，查询速度快。需要定期更新locate时刻。

    注意：第一次运行前，必须使用updatedb创建locate数据库

    $ updatedb
$ locate hello.txt

  3. which：查看某个指令在哪个目录下

    which ls

  4. grep [选项] 查找内容 源文件：过滤查找。

    常用选项：-n 显示匹配行及行号。-i 忽略字母大小写。

    cat /home/hello.txt | grep -n "yes"：查找yes所在行，并且显示行号。

    grep -n "yes" /home/hello.txt：同上

压缩和解压类

  1. gzip/gunzip：前一个用于压缩，后一个用于解压

    gzip 文件 ：压缩文件，将文件压缩为*.gz文件

第十章 实操篇-组管理和权限管理

概念

每个用户必须属于一个组，不能独立于组之外。在Linux中，每个文件有所有者、所在组、其他组的概念。

  1. 所有者。这个文件是谁的，谁创建的。
  2. 所在组。所在者所在的组。
  3. 其他组。除了所在组，其他那些对于这个文件都是其他组。

ls -ahl：查看文件夹信息的指令中有所有者和所在组的信息。

chown 用户名 文件名：修改文件所有者。

chgrp 组名 文件名：修改文件所在的组。即change group

组的创建

groupadd 组名：创建组

useradd -g 用户组 用户名：新增用户，并添加到指定用户组。不指定的话默认开一个同名的组给新用户。

修改用户所在组

usermod -g 新组 用户名：修改用户所在的组

usermod -d 目录名 用户名：修改用户登陆的初始目录。注意用户需要有进入新目录的权限。

权限的基本介绍

ls -l：指令打印起始显示了一串字母，一共10位。

  • 第0位：确定文件类型

    • l 表示链接，相当于快捷方式
    • d 表示目录，相当于文件夹
    • c 表示字符设备文件，鼠标，键盘
    • b 表示块设备，比如磁盘
    • - 表示普通文件

  • 第1-3位：所有者拥有的对该文件的权限
    +

  • 第4-6位：所属组也即同组的用户拥有的该文件的权限

  • 第7-9位：其他用户拥有的该文件的权限

其中，三位字符代表三个权限的有无，含义如下：

  • 对应文件的权限：
    • r 可读。可以读取，查看
    • w 可写。可以修改，但是不代表可以删除。删除一个文件的前提是对该文件所在的目录有写权限
    • x 可执行（execute）。可以被执行
  • 对应文件夹的权限：
    • r 可读。可以读取，ls 查看目录内容
    • w 可写。可以修改，对目录内创建+删除+重命名目录
    • x 可执行（execute）。可以进入该目录

修改文件所有者

chown newowner 文件/目录：改变所有者

chown newowner:newgroup 文件/目录：改变所有者和所在组

-R 如果是目录，则使其下所有子文件或目录递归生效

修改权限

chmod：修改文件或者目录的权限

文件的权限和所在文件夹的权限是分离的。有对文件的权限不一定能进所在文件夹

如果要对目录内的文件进行操作，需要先拥有对该目录的相应权限

  1. 第一种方式：+、-、=变更权限

    u代表所有者，g代表所有组，o代表其他人，a代表所有人（u、g、o的总和）

    举例：

    chmod u=rwx,g=rx,o=rx abc.txt：给abc文件的所有者读写执行权限，所在组读和执行权限，给其他人读和执行权限。

    chmod u-x,g+w abc：给abc文件的所有者除去执行权限，所在组增加写权限

    chmod a+r abc：给所有用户增加abc文件的读权限

  2. 通过数字变更权限

    r = 4， w = 2, x = 1。rwx = 4+2+1 = 7

    chmod u=rwx,g=rx,o=x abc相当于chmod 751 abc

对文件夹的rwx权限的细节讨论：

x：表示可以进入到该目录，比如cd进入

r：表示可以ls，将目录的内容列举显示出来

w：表示在该目录删除、创建、重命名文件

第十二章 实操篇-Linux磁盘分区、挂载

Linux分区

基本原理：

Linux无论几个分区，分给哪一目录使用，归根结底就只有一个根目录，一个独立且唯一的文件结构，Linux的每个分区都是用来组成整个文件系统的一部分。

Linux采用了一种“载入”的方法，整个文件系统包含了一整套文件和目录，将一个分区和一个目录联系起来。这时要载入的一个分区将使它的存储空间在一个目录下获得。

硬盘说明：

  • Linux硬盘分IDE硬盘和SCSI硬盘，目前基本上是SCSI硬盘
  • 对于IDE硬盘，驱动器标识符为hdx~,其中”hd”表明分区所在设备的类型，这里是指IDE硬盘了。“x”为盘号(有四种： a为基本盘，b为基本从属盘, c为辅助主盘，d为辅助从属盘),”~”代表分区，前四个分区用数字1到4表示，它们是主分区或扩展分区，从5开始就是逻辑分区，例，hda3表示为第一个IDE硬盘上的第三个主分区或扩展分区,hdb2表示为第二个IDE硬盘上的第二个主分区或扩展分区。
  • 对于SCSI更盘则标识为”sdx~” ， SCSI硬盘是用”sd” 来表示分区所在设备的类型的，其余则和IDE硬盘的表示方法一样。

第十四章 实操篇-进程管理

基本介绍

在LINUX中，每个执行的程序都称为个进程。每一个进程都分配一个ID号(pid,进程号)。每个进程都可能以两种方式存在的。前台与后台，所谓前台进程就是用户目前的屏幕上可以进行操作的。后台进程则是实际在操作,但由于屏幕上无法看到的进程,通常使用后台方式执行。一般系统的服务都是以后台进程的方式存在,而且都会常驻在系统中。直到关机才才结束。

服务的运行级别：

常用的是级别3和5
运行级别0 :系统停机状态,系统默认运行级别不能设为0 ,否则不能正常启动
运行级别1 :单用户工作状态，root权限，用于系统维护，禁止远程登陆
运行级别2 :多用户状态(没有NFS) ,不支持网络

运行级别4 :系统未使用,保留
运行级别5 : 登陆后进入图形GUI模式
运行级别6 :系统正常关闭并重启，默认运行级别不能设为6 ,否则不能正常启动

  2. chkconfig指令管理的服务在/etc/init.d查看

  3. 注意: Centos7.0后，很多服务使用systemctl管理

chkconfig基本语法:

  • 查看服务chkconfig --list [| grep xxx]

  • chkconfig 服务名 --list

  • chkconfig --level 5 服务名 on/off

chkconfig重新设置服务后自启动或关闭，需要重启机器reboot生效

systemctl管理命令：

  1. systemctl [start|stop|restart|reload|status] 服务名
  2. service指令管理的服务在/usr/lib/systemd/system用ls -l /usr/lib/systemd/system/查看

systemctl设置服务的自启动状态

  1. systemctl list-unit-files [| grep 服务名] 查看服务开机启动状态，grep可以进行过滤
  2. systemctl enable 服务名设置服务开机启动
  3. systemctl disable 服务名关闭服务开机启动
  4. systemctl is-enabled 服务名查询某个服务是否是自启动的

其中第二三条是针对运行级别3和5的。因为CentOS7.0之后特性，所以没有指定特定运行级别的相关指令。

案例：

查看当前防火墙状态，关闭和开启防火墙。firewalld.service

systemctl status firewalld.service全称

systemctl stop firewalld简写也可

systemctl start firewalld

注意：

  1. 关闭或者启用防火墙后，立即生效
  2. 这种方式(指stop 和start )是临时生效，重启系统后会回归以前对服务的设置
  3. 如果希望设置某个服务自启动或关闭永久生效，要使用systemctl [enable|disable] 服务名

打开或关闭指定端口：

实际生产中，一般是把防火墙打开的，但是要把指定端口打开用于服务。

防火墙指令：

  1. 打开端口: firewall-cmd --permanent --add-port=端口号/协议
  2. 关闭端口: firewall-cmd --permanent --remove-port=端口号/协议
  3. 重新载入，才能生效: firewall-cmd --reload
  4. 查询端口是否开放: firewall-cmd --query-port=端口/协议

端口与协议怎么查：

netstat -anp | more 查看端口信息

案例：

  1. 启动防火墙，测试111端口是否能telnet。答案：不行

  2. 开放111端口。

    firewall-cmd --permanent --add-port=111/tcp

    firewall-cmd --reload

  3. 关闭111端口。

    firewall-cmd --permanent --remove-port=111/tcp

    firewall-cmd --reload

动态监控进程

介绍：

top与ps命令相似，都用来显示正在执行的进程。最大的不同之处在于top在执行一段时间可以更新正在运行的进程。

基本语法：

top [选项]

选项 含义
-d 秒数 指定top命令每隔几秒更新。默认为3秒
-i 使top不显示任何闲置或者僵死进程
-p 通过指定监控进程ID来仅仅监控某个进程的状态

示例：

分析：

第一行：系统时间、已运行时间、用户数、负载值（如果三个值平均值大于0.7说负载比较大，要进行优化）

第二行：任务数、各个状态分别的任务数

第三行：用户、系统、ni（？）、id（？）分别占用CPU比例

第四行：内存占用数量

第五行：swap分区占用情况

交互操作说明：

操作 功能
P 以CPU使用率排序，默认就是此项
M 以内存使用率排序
N 以PID排序
q 退出top

案例：

  1. 监视特定用户

    top 回车，输入u 回车，再输入用户名

  2. 终止指定的进程

    top回车，输入k回车，再输入要结束的ID号

  3. 指定系统状态更新的时间（每隔10秒自动更新）默认是3秒

    top -d 10

监控网络状态

查看系统网络情况：

基本语法：netstat [选项]

选项说明：-an 按照一定的顺序输出。-p 显示哪个进程在调用

案例：

查看服务名未sshd的服务的信息 netstat -anp | grep sshd

检测主机连接命令：

基本语法：ping ip地址

rpm包管理

rpm用户互联网下载包的打包及安装工具，包含在某些linux分发版中。生成.rpm文件是RedHat Package Manner缩写，类似windows系统的setup.exe

查询命令：

rpm -qa 查询已安装的rpm列表。rpm -qa | grep xxxx

rpm包名的基本格式：

firefox- 60.2.2-1.el7 .centos.x8664

名称、版本号、适用操作系统（表示centos7.x的64位系统，如果是1686、i386表示32位系统, noarch表示通用。）

yum list | grep xx软件列表 查询yum服务器是否有需要安装的软件

第十七章 大数据定制篇-shell编程

介绍：

实际生产环境需要写shell脚本进行程序或服务器维护，管理集群等。

shell是一个命令行解释器，为用户提供了一个想linux内核发送请求以便运行程序的界面系统级程序，用户可以用shell来启动、挂起、停止甚至是编写一些程序。

Shell脚本的执行方式

脚本格式要求：

  1. 脚本里面的内容规定以#!/bin/bash开头
  2. 脚本需要有可执行权限 chmod u+x hello.sh

脚本的常用执行方式

方式一：输入脚本的绝对路径或者相对路径

注意首先赋予.sh脚本的执行权限，再执行./hello.sh或使用绝对路径

方式二：sh 脚本

不用赋予脚本执行权限，直接执行即可

例如：sh hello.sh 也可以使用绝对路径

Shell的变量

分为两种，系统变量和用户自定义变量

系统变量：$HOME，$PWD，$USER等。

显示当前shell中所有变量：set

变量定义：

定义变量：变量名=值

撤销变量：unset 变量名

案例：

定义变量A 。销毁变量A。声明静态的变量B=2，不能unset

#!/bin/bash
A=100
echo A=$A
echo "A=$A"
#两者一样，输出：A=100
unset A
echo "A=$A"
#输出：A=
readonly B=2
echo "B=$B"
#输出：B=2
#如果写unset B 运行会报错

变量定义规则：

  1. 变量名称可以由字母、数字和下划线组成,但是不能以数字开头。
  2. 等号两侧不能有空格
  3. 变量名称一般习惯为大写。默认规范

将命令的返回值赋给变量：

  1. A=`date` 反引号，运行里面的**命令,**并把结果返回给变量A 。没有反引号的话是把date这个单词给A
  2. A=$(date)等价于反引号

设置环境变量

基本语法：

  1. export 变量名=变量值 这句话写在/etc/profile ，将shell变量输出为环境变量/全局变量。可供其他.sh脚本里面使用
  2. source /etc/profile注意刷新，让新定义的配置信息生效
  3. 在脚本中就可以使用1中定义的环境变量

shell脚本多行注释：

:<<!

注释行1

注释行2

!

位置参数变量

介绍：

简单讲就是在脚本里面获取以后调用该脚本时命令行输入的参数信息。./myshell.sh 100 200

基本语法：

语句 功能
$n n为数字，$0代表命令本身，$1-$9代表第1-9个参数。十个以上的参数用大括号包含。形如${10}
$* 代表命令行中的所有参数，把所有的参数看成一个整体
$@ 同上，但把每个参数区分对待
$# 代表命令行中所有参数的个数

案例：

#!/bin/bash
echo "0=$0 1=$1 2=$2"
echo "所有=$*"
echo "$@"
echo "参数的个数=$#"

预定义变量

介绍：

shell设计者事先设计好的变量，可以在shell脚本中使用

基本语法：

语句 功能
$$ 当前执行的进程id，也即执行该脚本的进程
$! 最后一个后台方式运行的进程id
$? 最后一次执行的命令的返回状态。如果这个变量值为0，说明上一个命令正确执行。如果非零，则不正确

脚本内调用其他脚本：

/root/shcode/myshell.sh & 以后台的方式运行一个脚本，并获取他的进程号

运算符

基本语法：

  1. $((运算式))或者$[运算式]或者``expr m + n`
  2. 注意expr运算符间要有空格。如果没有空格，整体当字符串
  3. expr 3 - 2 其中运算符可以为\*,/,%

案例：

#!/bin/bash
RES1=$(((2+3)*4))
echo res1=$RES1
RES2=$[(2+3)*4]
echo res2=$RES2
TEMP=`expr 2 + 3`
RES3=`expr $TEMP \* 4`
echo RES3=$RES3
SUM=$[$1+$2]
echo SUM=$SUM

条件判断

[ condition ] 注意condition前后都有空格。注意：非空返回true。[ abc ]为true

[ condition ] && echo OK || noOK条件满足执行后面的语句

常用判断条件：

  1. = 字符串比较
  2. 两个整数的比较
    • -lt 小于 -le 小于等于 little equal
    • -gt 大于 -ge 大于等于
    • -ne 不等于
  3. 按照文件权限进行判断
    • -r / -w / -x 有读/写/执行权限
  4. 按照文件类型进行判断
    • -f 文件存在且是一个常规的文件
    • -e 文件存在
    • -d 文件存在且是一个目录
#!/bin/bash
#案例1 ”OK“是否等于”OK“
if [ "OK" = "OK" ] then
    echo "equal"
fi
#案例2 23是否大于等于22
if [ 23 -ge 22 ] then
    echo "greater equal"
fi
#案例3 /root/shcode/aaa.txt 目录中的文件是否存在
if [ -f /root/shcode/aaa.txt ] then
    echo "存在"
fi
if [ abc ] then
    echo "会输出"
fi
if [] then
    echo ".."
fi
#会有问题
if [ ] then
    echo ".."
fi
#没问题 且判断为false

if [ condition ]
then
#代码
fi

then
#代码
elif [ condition ]
#代码
fi
